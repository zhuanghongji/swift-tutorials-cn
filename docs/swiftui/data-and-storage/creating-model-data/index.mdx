## 监控应用程序中的数据变化

将特定于视图的数据封装在应用程序的视图层次结构中，以使你的视图可重用。

### 概述

你通常使用与应用程序的用户界面（UI）和其他逻辑分开的数据模型，来存储和处理应用程序中的数据。这种分离促进了模块化，提高了可测试性，并使你更容易推断应用程序的工作原理。

传统上，你使用视图控制器在模型和 UI 之间来回移动数据（move data back and forth），但 SwiftUI 会为你处理大部分同步。要在数据更改时更新视图，你可以将数据模型类设置为可观察对象，发布它们的属性，并使用属性包装器声明它们的实例。为了确保用户驱动的数据更改回流到模型中，你可以将 UI 控件绑定到模型属性。这些功能协同工作，可帮助你维护数据的单一事实来源。

:::caution

本文介绍了如何在 "最低部署目标早于 iOS 17、iPadOS 17、macOS 14、tvOS 17 或 watchOS 10 的 SwiftUI 应用程序" 中观察模型数据更改。为其中之一的较新版本操作系统准备你的应用程序，请参阅 [从 Observable 对象协议迁移到 Observable 宏](https://developer.apple.com/documentation/swiftui/migrating-from-the-observable-object-protocol-to-the-observable-macro)。如果你的应用程序的最低部署目标是 iOS 17、iPadOS 17、macOS 14、tvOS 17 或 watchOS 10 或更高版本，请使用 [管理应用程序中的模型数据](https://developer.apple.com/documentation/swiftui/managing-model-data-in-your-app) 中描述的观察方法。

:::

### 使模型数据可观察

要使模型中的数据更改对 SwiftUI 可见，请对模型类采用 [ObservableObject](https://developer.apple.com/documentation/Combine/ObservableObject) 协议。例如，你可以创建一个作为可观察对象的 `Book` 类：

```swift
class Book: ObservableObject {
}
```

系统自动推断类的 [ObjectWillChangePublisher](https://developer.apple.com/documentation/Combine/ObservableObject/ObjectWillChangePublisher) 关联类型，并合成所需的 [objectWillChange](https://developer.apple.com/documentation/Combine/ObservableObject/objectWillChange-2oa5v) 方法，该方法发出 "已发布属性（published properties）" 的更改值。要发布属性，请将 [Published](https://developer.apple.com/documentation/Combine/Published) 属性包装器添加到属性的声明中：

```swift
class Book: ObservableObject {
    @Published var title = "Sample Book Title"
}
```

当你不需要已发布的属性时，避免它的开销。仅发布可以更改且对 UI 重要的属性。例如， `Book` 类可能有一个 `identifier` 属性，该属性在初始化后永远不会改变：

```swift
class Book: ObservableObject {
    @Published var title = "Sample Book Title"

    // highlight-next-line
    let identifier = UUID() // A unique identifier that never changes.
}
```

你仍然可以在用户界面中显示 `identifier`，但由于它并非 "可发布的（published）"，所以 SwiftUI 不会监视该特定属性的更改。

### 监控可观察对象的变化

要告诉 SwiftUI 监视可观察对象，请将 [ObservedObject](https://developer.apple.com/documentation/swiftui/observedobject) 属性包装器添加到属性的声明中：

```swift
struct BookView: View {
    // highlight-next-line
    @ObservedObject var book: Book

    var body: some View {
        Text(book.title)
    }
}

```

你可以将观察对象的各个属性传递给子视图，如上所示。当数据发生变化时，例如从磁盘加载新数据时，SwiftUI 会更新所有受影响的视图。你还可以将整个可观察对象传递给子视图，并跨视图层次结构的级别共享模型对象：

```swift
struct BookView: View {
    @ObservedObject var book: Book

    var body: some View {
        BookEditView(book: book)
    }
}

struct BookEditView: View {
    @ObservedObject var book: Book

    // ...
}
```

### 在视图中实例化模型对象

SwiftUI 可能随时创建或重新创建视图，所以使用 "给定输入集（a given set of inputs）" 初始化视图始终会产生相同的视图，这一点很重要。因此，在视图内创建观察对象是不安全的。相反，SwiftUI 提供了 [StateObject](https://developer.apple.com/documentation/swiftui/stateobject) 属性包装器，它为存储在视图层次结构中的引用类型创建 "单一事实来源"。你可以通过以下方式在视图中安全地创建 `Book` 实例：

```swift
struct LibraryView: View {
    // highlight-next-line
    @StateObject private var book = Book()

    var body: some View {
        BookView(book: book)
    }
}
```

状态对象的行为类似于观察对象，不同之处在于 SwiftUI 为给定视图实例创建和管理单个对象实例，无论它重新创建视图多少次。你可以在本地使用该对象，或者将状态对象传递到另一个视图的观察对象属性中，如上面的示例所示。

虽然 SwiftUI 不会在视图中重新创建状态对象，但它会为每个视图实例创建一个不同的对象实例。例如，以下代码中的每个 `LibraryView` 都会获得一个唯一的 `Book` 实例：

```swift
VStack {
    LibraryView()
    LibraryView()
}
```

你还可以在顶级 [App](https://developer.apple.com/documentation/swiftui/app) 实例或应用程序的 [Scene](https://developer.apple.com/documentation/swiftui/scene) 实例之一中创建状态对象。例如，如果你定义一个名为 `Library` 的可观察对象来保存 "图书阅读器应用程序" 的书籍集合，则可以在应用程序的顶级结构中创建一个 `Library` 实例：

```swift
@main
struct BookReader: App {
    // highlight-next-line
    @StateObject private var library = Library()

    // ...
}
```

### 在整个应用程序中共享对象

如果你想要在整个应用程序中使用一个数据模型对象，但不想在多个层次结构见传递，可以使用 [environmentObject(\_:)](<https://developer.apple.com/documentation/swiftui/view/environmentobject(_:)>) 视图修饰器将该对象放入 environment 中：

```swift
@main
struct BookReader: App {
    @StateObject private var library = Library()

    var body: some Scene {
        WindowGroup {
            LibraryView()
                // highlight-next-line
                .environmentObject(library)
        }
    }
}
```

应用修饰器的视图的任何后代视图（descendant view）都可以通过使用 [EnvironmentObject](https://developer.apple.com/documentation/swiftui/environmentobject) 属性包装器声明属性来访问数据模型实例：

```swift
struct LibraryView: View {
    @EnvironmentObject var library: Library

    // ...
}
```

如果你使用环境对象，则可以将其添加到应用程序层次结构顶部的视图中，如上所示。或者，你可以将其添加到视图层次结构中子树的根视图中。无论哪种方式（Either way），请记住还将其添加到 "使用该对象" 或 "具有使用该对象的后代的任何视图" 的预览提供程序中：

```swift
struct LibraryView_Previews: PreviewProvider {
    static var previews: some View {
        LibraryView()
            // highlight-next-line
            .environmentObject(Library())
    }
}
```

### 使用绑定创建双向连接

当你允许某人更改 UI 中的数据时，请使用相应属性（corresponding property）的绑定。这确保更改能够自动流回到数据模型。你可以通过在对象名称前添加美元符号（`$`）来获取 "观察对象"、"状态对象" 或 "环境对象" 属性的绑定。例如，如果你让某人通过将 [TextField](https://developer.apple.com/documentation/swiftui/textfield) 添加到 `BookEditView` 来编辑书籍的标题，请为 text field 提供与书籍的 `title` 属性的绑定:

```swift
struct BookEditView: View {
    @ObservedObject var book: Book

    var body: some View {
        // highlight-next-line
        TextField("Title", text: $book.title)
    }
}
```

绑定将视图元素连接到底层模型，以便人们可以直接对模型数据进行更改。

## StateObject

实例化可观察对象的属性包装类型。

```swift
@frozen @propertyWrapper public struct StateObject<ObjectType> : DynamicProperty
        where ObjectType : ObservableObject
{
    // 创建一个具有初始包装值的新状态对象
    @inlinable public init(wrappedValue thunk: @autoclosure @escaping () -> ObjectType)

    // 状态对象引用的基础值
    @MainActor public var wrappedValue: ObjectType { get }

    // 状态对象的影射（projection），创建与其属性的绑定
    @MainActor public var projectedValue: ObservedObject<ObjectType>.Wrapper { get }
}
```

### 概述

使用状态对象作为 "存储在视图层次结构中的" 引用类型的 "单一事实来源"。通过将 `@StateObject` 属性应用于属性声明并提供初始值，在 [App](https://developer.apple.com/documentation/swiftui/app)、 [Scene](https://developer.apple.com/documentation/swiftui/scene) 或 [View](https://developer.apple.com/documentation/swiftui/view) 中创建状态符合 [ObservableObject](https://developer.apple.com/documentation/swiftui/view) 协议的对象。将状态对象声明为私有，以防止从成员初始化器设置它们，因为这可能与 SwiftUI 提供的存储管理冲突：

```swift
class DataModel: ObservableObject {
    @Published var name = "Some Name"
    @Published var isEnabled = false
}

struct MyView: View {
    @StateObject private var model = DataModel() // Create the state object.

    var body: some View {
        Text(model.name) // Updates when the data model changes.
        MySubView()
            .environmentObject(model)
    }
}
```

在声明状态对象的容器的生命周期内，SwiftUI 仅创建一次模型对象的新实例。例如，如果视图的输入发生更改，SwiftUI 不会创建新实例，但如果视图的标识发生更改，则会创建新实例。当可观察对象的已发布（published）属性发生更改时，SwiftUI 会更新依赖于这些属性的任何视图，例如上例中的 `Text` 视图。

:::note

如果需要存储值类型（例如结构体、字符串或整数），请改用 [State](https://developer.apple.com/documentation/swiftui/state) 属性包装器。如果需要存储符合 [Observable()](<https://developer.apple.com/documentation/Observation/Observable()>) 协议的引用类型，也可以使用 [State](https://developer.apple.com/documentation/swiftui/state) 。要了解有关 SwiftUI 中更多关于 Observation 的信息，请参阅 [管理应用程序中的模型数据](https://developer.apple.com/documentation/swiftui/managing-model-data-in-your-app)。

:::

### 与子视图共享状态对象

你可以通过具有 [ObservedObject](https://developer.apple.com/documentation/swiftui/observedobject) 标注（attribute）的属性（property）将状态对象传递到子视图中。或者，通过将 [environmentObject(\_:)](<https://developer.apple.com/documentation/swiftui/view/environmentobject(_:)>) 修饰器应用于视图，将对象添加到视图层次结构的环境中，如上面代码中的 `MySubView` 。然后，你可以在 `MySubView` 或其任何子视图中使用 [EnvironmentObject](https://developer.apple.com/documentation/swiftui/environmentobject) 标注（attribute）读取对象：

```swift
struct MySubView: View {
    // highlight-next-line
    @EnvironmentObject var model: DataModel

    var body: some View {
        Toggle("Enabled", isOn: $model.isEnabled)
    }
}
```

使用美元符号 `$` 运算符获取状态对象属性的 [Binding](https://developer.apple.com/documentation/swiftui/binding)。当你想要创建双向连接时，请使用绑定。在上面的代码中，[Toggle](https://developer.apple.com/documentation/swiftui/toggle) 通过绑定来控制模型的 `isEnabled` 值。

### 使用外部数据初始化状态对象

当状态对象的初始状态取决于来自其容器外部的数据时，你可以从其容器的初始化器内显式调用该对象的初始化器。例如，假设上一个示例中的数据模型在初始化期间采用 name 输入，并且你希望使用来自视图外部的该名称的值。你可以通过在为视图创建的显式初始化程序内调用状态对象的初始化程序来完成此操作：

```swift
struct MyInitializableView: View {
    // highlight-start
    @StateObject private var model: DataModel

    init(name: String) {
        // SwiftUI ensures that the following initialization uses the
        // closure only once during the lifetime of the view, so
        // later changes to the view's name input have no effect.
        _model = StateObject(wrappedValue: DataModel(name: name))
    }
    // highlight-end

    var body: some View {
        VStack {
            Text("Name: \(model.name)")
        }
    }
}
```

执行此操作时请务必小心。 SwiftUI 仅在你第一次在给定视图中调用其初始值器时初始化状态对象。这确保了，即使视图的输入发生变化，对象也能提供稳定的存储。但是，如果显式初始化状态对象，可能会导致意外行为或不需要的副作用（side effect）。

在上面的示例中，如果 `MyInitializableView` 的 `name` 输入发生更改，SwiftUI 将使用新值重新执行视图的初始化器。但是，SwiftUI 仅在你第一次调用状态对象的初始化器时，执行你提供给状态对象的初始化器的自动闭包（autoclosure），因此模型存储的 `name` 值不会更改。

当 "对象所依赖的外部数据" 对于 "对象容器的给定实例" 没有更改时，显式的 "状态对象初始化" 效果很好（works well）。例如，你可以创建两个具有不同常量名称的视图：

```swift
var body: some View {
    VStack {
        MyInitializableView(name: "Ravi")
        MyInitializableView(name: "Maria")
    }
}
```

:::caution

即使对于可配置的状态对象，你仍然将其声明为私有。这确保你不会意外地通过视图的成员初始化器设置参数，因为这样做可能会与框架的存储管理发生冲突并产生意外的结果。

:::

### 通过更改视图标识强制重新初始化

如果你希望 SwiftUI 在视图输入更改时重新初始化状态对象，请确保视图的标识同时更改。实现此目的的一种方法是使用 [id(\_:)](<https://developer.apple.com/documentation/swiftui/view/id(_:)>) 修饰器将视图的标识绑定到更改的值。

```swift
extension View {

    /// Binds a view's identity to the given proxy value.
    ///
    /// When the proxy value specified by the `id` parameter changes, the
    /// identity of the view — for example, its state — is reset.
    @inlinable public func id<ID>(_ id: ID) -> some View where ID : Hashable
}
```

例如，你可以确保 `MyInitializableView` 实例的标识在其 `name` 输入更改时发生更改：

```swift
MyInitializableView(name: name)
    .id(name) // Binds the identity of the view to the name property.
```

:::note

如果你的视图出现在 [ForEach](https://developer.apple.com/documentation/swiftui/foreach) 内，它会隐式接收使用相应数据元素的标识符的 `id(_:)` 修饰器。

:::

如果你需要视图根据多个值的更改重新初始化状态，你可以使用 [Hasher](https://developer.apple.com/documentation/Swift/Hasher) 将这些值组合成一个标识符。例如，如果你希望在 `name` 或 `isEnabled` 的值发生更改时更新 `MyInitializableView` 中的数据模型，则可以将这两个变量合并到单个哈希中:

```swift
var hash: Int {
    var hasher = Hasher()
    hasher.combine(name)
    hasher.combine(isEnabled)
    return hasher.finalize()
}
```

然后将组合哈希（combined hash）作为标识符应用到视图：

```swift
MyInitializableView(name: name, isEnabled: isEnabled)
    // highlight-next-line
    .id(hash)
```

请注意（Be mindful of）每次输入更改时，重新初始化状态对象的性能成本（the performance cost）。此外，更改视图标识可能会产生副作用。例如，如果视图的标识同时发生变化，SwiftUI 不会自动为 "视图内的更改" 设置动画。此外，更改标识会重置视图持有的所有状态，包括你管理为 [State](https://developer.apple.com/documentation/swiftui/state)、[FocusState](https://developer.apple.com/documentation/swiftui/focusstate)、[GestureState](https://developer.apple.com/documentation/swiftui/gesturestate) 等的值。

## ObservableObject

可观察对象：一种具有发布者的对象类型，该发布者在对象更改之前发出。

> `ObservableObject` 是 `Protocol` 类型，来自 [Combile](https://developer.apple.com/documentation/combine) 框架。

```swift
public protocol ObservableObject : AnyObject {

    // 在对象更改之前发出的发布者类型
    associatedtype ObjectWillChangePublisher : Publisher = ObservableObjectPublisher
            where Self.ObjectWillChangePublisher.Failure == Never

    // 在对象更改之前发出的发布者
    var objectWillChange: Self.ObjectWillChangePublisher { get }
}

final public class ObservableObjectPublisher : Publisher {

    // 该发布者发布的值类型
    public typealias Output = Void

    // 该发布者可能发布的错误类型
    public typealias Failure = Never

    // 创建一个可观察对象发布者实例
    public init()

    // 将指定订阅者附加到此发布者
    final public func receive<S>(subscriber: S)
            where S : Subscriber, S.Failure == Never, S.Input == ()

    // 将更改后的值发送给下游订阅者
    final public func send()
}
```

### 概述

默认情况下，`ObservableObject` 合成一个 `objectWillChange` 发布者，该发布者在其任何 `@Published` 属性更改之前发出更改的值。

```swift
class Contact: ObservableObject {
    @Published var name: String
    @Published var age: Int

    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }

    func haveBirthday() -> Int {
        age += 1
        return age
    }
}

let john = Contact(name: "John Appleseed", age: 24)
cancellable = john.objectWillChange
    .sink { _ in
        print("\(john.age) will change")
}
print(john.haveBirthday())

// Prints "24 will change"
// Prints "25"
```

## ObservedObject

观察对象：一种属性包装类型，它订阅 "可观察对象" 并在 "可观察对象" 发生更改时使视图无效。

```swift
@propertyWrapper @frozen public struct ObservedObject<ObjectType> : DynamicProperty
        where ObjectType : ObservableObject
{
    // 底层可观察对象的包装器，可以创建对其属性的绑定
    @dynamicMemberLookup @frozen public struct Wrapper {

        // 获取到指定 key path 的值的绑定
        public subscript<Subject>(
            dynamicMember keyPath: ReferenceWritableKeyPath<ObjectType, Subject>
        ) -> Binding<Subject> { get }
    }

    // 创建具有初始值的观察对象
    public init(initialValue: ObjectType)

    // 创建具有初始包装值的观察对象
    public init(wrappedValue: ObjectType)

    // 被观察对象引用的基础值（underlying value）
    @MainActor public var wrappedValue: ObjectType

    // 观察对象的投影（projection），创建与其属性的绑定
    @MainActor public var projectedValue: ObservedObject<ObjectType>.Wrapper { get }
}
```

### 概述

当你的输入是 [ObservableObject](https://developer.apple.com/documentation/Combine/ObservableObject) 且希望视图在对象的 "已发布属性" 更改时更新，可以将 `@ObservedObject` 标注添加到 SwiftUI [View](https://developer.apple.com/documentation/swiftui/view) 的参数中。你通常这样做，是为了将 [StateObject](https://developer.apple.com/documentation/swiftui/stateobject) 传递到子视图中。

以下示例将数据模型定义为可观察对象，在视图中将模型实例化为状态对象，然后将该实例作为观察对象传递到子视图：

```swift
class DataModel: ObservableObject {
    @Published var name = "Some Name"
    @Published var isEnabled = false
}

struct MyView: View {
    // highlight-next-line
    @StateObject private var model = DataModel()

    var body: some View {
        Text(model.name)
        // highlight-next-line
        MySubView(model: model)
    }
}

struct MySubView: View {
    // highlight-next-line
    @ObservedObject var model: DataModel

    var body: some View {
        Toggle("Enabled", isOn: $model.isEnabled)
    }
}

```

当可观察对象的任何 "已发布属性" 发生更改时，SwiftUI 会更新依赖于该对象的任何视图。子视图还可以对模型属性进行更改，例如上面示例中的 [Toggle](https://developer.apple.com/documentation/swiftui/toggle)，这些属性会传播（propagate）到 "整个视图层次结构" 中的其他观察者。

:::note

不要用 `@ObservedObject` 包装符合 [Observable](https://developer.apple.com/documentation/Observation/Observable) 协议的对象。SwiftUI 自动跟踪 `body` 中使用的 `Observable` 对象的依赖关系，并在数据更改时更新依赖视图。尝试用 `@ObservedObject` 包装 [Observable](https://developer.apple.com/documentation/Observation/Observable) 对象可能会导致编译器错误，因为它要求其包装的对象符合 [ObservableObject](https://developer.apple.com/documentation/Combine/ObservableObject) 协议。

如果视图需要其 `body` 中的属性绑定到 `Observable` 对象，请使用 [Bindable](https://developer.apple.com/documentation/swiftui/bindable) 属性包装器包装该对象；例如， `@Bindable var model: DataModel`。有关更多信息，请参阅 [管理应用程序中的模型数据](https://developer.apple.com/documentation/swiftui/managing-model-data-in-your-app)。

:::

## 总结

`ObservableObject` 的使用是比较复杂的，如果最低目标版本是 iOS 17 时，可直接使用 `Observable` 代替。
