创建模型数据

创建模型数据

原文：https://developer.apple.com/documentation/swiftui/model-data#creating-model-data


监控应用程序中的数据变化​

将特定于视图的数据封装在应用程序的视图层次结构中，以使你的视图可重用。

概述​

你通常使用与应用程序的用户界面（UI）和其它逻辑分开的数据模型，来存储和处理应用程序中的数据。这种分离促进了模块化，提高了可测试性，并使你更容易推断应用程序的工作原理。

传统上，你使用视图控制器在模型和 UI 之间来回移动数据（move data back and forth），但 SwiftUI 会为你处理大部分同步。要在数据更改时更新视图，你可以将数据模型类设置为可观察对象，发布它们的属性，并使用属性包装器声明它们的实例。为了确保用户驱动的数据更改回流到模型中，你可以将 UI 控件绑定到模型属性。这些功能协同工作，可帮助你维护数据的单一事实来源。

警告

本文介绍了如何在 "最低部署目标早于 iOS 17、iPadOS 17、macOS 14、tvOS 17 或 watchOS 10 的 SwiftUI 应用程序" 中观察模型数据更改。为其中之一的较新版本操作系统准备你的应用程序，请参阅 从 Observable 对象协议迁移到 Observable 宏。如果你的应用程序的最低部署目标是 iOS 17、iPadOS 17、macOS 14、tvOS 17 或 watchOS 10 或更高版本，请使用 管理应用程序中的模型数据 中描述的观察方法。

使模型数据可观察​

要使模型中的数据更改对 SwiftUI 可见，请对模型类采用 ObservableObject 协议。例如，你可以创建一个作为可观察对象的 Book 类：

class Book: ObservableObject {
}

系统自动推断类的 ObjectWillChangePublisher 关联类型，并合成所需的 objectWillChange 方法，该方法发出 "已发布属性（published properties）" 的更改值。要发布属性，请将 Published 属性包装器添加到属性的声明中：

class Book: ObservableObject {
    @Published var title = "Sample Book Title"
}

当你不需要已发布的属性时，避免它的开销。仅发布可以更改且对 UI 重要的属性。例如， Book 类可能有一个 identifier 属性，该属性在初始化后永远不会改变：

class Book: ObservableObject {
    @Published var title = "Sample Book Title"

    let identifier = UUID() // A unique identifier that never changes.
}

你仍然可以在用户界面中显示 identifier，但由于它并非 "可发布的（published）"，所以 SwiftUI 不会监视该特定属性的更改。

监控可观察对象的变化​

要告诉 SwiftUI 监视可观察对象，请将 ObservedObject 属性包装器添加到属性的声明中：

struct BookView: View {
    @ObservedObject var book: Book

    var body: some View {
        Text(book.title)
    }
}

你可以将观察对象的各个属性传递给子视图，如上所示。当数据发生变化时，例如从磁盘加载新数据时，SwiftUI 会更新所有受影响的视图。你还可以将整个可观察对象传递给子视图，并跨视图层次结构的级别共享模型对象：

struct BookView: View {
    @ObservedObject var book: Book

    var body: some View {
        BookEditView(book: book)
    }
}

struct BookEditView: View {
    @ObservedObject var book: Book

    // ...
}

在视图中实例化模型对象​

SwiftUI 可能随时创建或重新创建视图，所以使用 "给定输入集（a given set of inputs）" 初始化视图始终会产生相同的视图，这一点很重要。因此，在视图内创建观察对象是不安全的。相反，SwiftUI 提供了 StateObject 属性包装器，它为存储在视图层次结构中的引用类型创建 "单一事实来源"。你可以通过以下方式在视图中安全地创建 Book 实例：

struct LibraryView: View {
    @StateObject private var book = Book()

    var body: some View {
        BookView(book: book)
    }
}

状态对象的行为类似于观察对象，不同之处在于 SwiftUI 为给定视图实例创建和管理单个对象实例，无论它重新创建视图多少次。你可以在本地使用该对象，或者将状态对象传递到另一个视图的观察对象属性中，如上面的示例所示。

虽然 SwiftUI 不会在视图中重新创建状态对象，但它会为每个视图实例创建一个不同的对象实例。例如，以下代码中的每个 LibraryView 都会获得一个唯一的 Book 实例：

VStack {
    LibraryView()
    LibraryView()
}

你还可以在顶级 App 实例或应用程序的 Scene 实例之一中创建状态对象。例如，如果你定义一个名为 Library 的可观察对象来保存 "图书阅读器应用程序" 的书籍集合，则可以在应用程序的顶级结构中创建一个 Library 实例：

@main
struct BookReader: App {
    @StateObject private var library = Library()

    // ...
}

在整个应用程序中共享对象​

如果你想要在整个应用程序中使用一个数据模型对象，但不想在多个层次结构见传递，可以使用 environmentObject(_:) 视图修饰器将该对象放入 environment 中：

@main
struct BookReader: App {
    @StateObject private var library = Library()

    var body: some Scene {
        WindowGroup {
            LibraryView()
                .environmentObject(library)
        }
    }
}

应用修饰器的视图的任何后代视图（descendant view）都可以通过使用 EnvironmentObject 属性包装器声明属性来访问数据模型实例：

struct LibraryView: View {
    @EnvironmentObject var library: Library

    // ...
}

如果你使用环境对象，则可以将其添加到应用程序层次结构顶部的视图中，如上所示。或者，你可以将其添加到视图层次结构中子树的根视图中。无论哪种方式（Either way），请记住还将其添加到 "使用该对象" 或 "具有使用该对象的后代的任何视图" 的预览提供程序中：

struct LibraryView_Previews: PreviewProvider {
    static var previews: some View {
        LibraryView()
            .environmentObject(Library())
    }
}

使用绑定创建双向连接​

当你允许某人更改 UI 中的数据时，请使用相应属性（corresponding property）的绑定。这确保更改能够自动流回到数据模型。你可以通过在对象名称前添加美元符号（$）来获取 "观察对象"、"状态对象" 或 "环境对象" 属性的绑定。例如，如果你让某人通过将 TextField 添加到 BookEditView 来编辑书籍的标题，请为 text field 提供与书籍的 title 属性的绑定:

struct BookEditView: View {
    @ObservedObject var book: Book

    var body: some View {
        TextField("Title", text: $book.title)
    }
}

绑定将视图元素连接到底层模型，以便人们可以直接对模型数据进行更改。

StateObject​

概述​

使用状态对象作为存储在视图层次结构中的引用类型的单一事实来源。通过将 @StateObject 属性应用于属性声明并提供初始值，在 App、 Scene 或 View 中创建状态对象符合 ObservableObject 协议。将状态对象声明为私有，以防止从成员初始化器设置它们，这可能与 SwiftUI 提供的存储管理冲突：

class DataModel: ObservableObject {
    @Published var name = "Some Name"
    @Published var isEnabled = false
}


struct MyView: View {
    @StateObject private var model = DataModel() // Create the state object.

    var body: some View {
        Text(model.name) // Updates when the data model changes.
        MySubView()
            .environmentObject(model)
    }
}

SwiftUI 在声明状态对象的容器的生命周期内仅创建一次模型对象的新实例。例如，如果视图的输入发生更改，SwiftUI 不会创建新实例，但如果视图的标识发生更改，则会创建新实例。当可观察对象的已发布属性发生更改时，SwiftUI 会更新依赖于这些属性的任何视图，例如上例中的 Text 视图。

备注

如果需要存储值类型（例如结构体、字符串或整数），请改用 State 属性包装器。如果需要存储符合 Observable() 协议的引用类型，也可以使用 State 。要了解有关 SwiftUI 中的观察的更多信息，请参阅 管理应用程序中的模型数据。

与子视图共享状态对象​

你可以通过具有 ObservedObject 属性的属性将状态对象传递到子视图中。或者，通过将 environmentObject(_:) 修饰器应用于视图，将对象添加到视图层次结构的环境中，如上面代码中的 MySubView 。然后，你可以使用 EnvironmentObject 属性读取 MySubView 或其任何后代中的对象：

struct MySubView: View {
    @EnvironmentObject var model: DataModel

    var body: some View {
        Toggle("Enabled", isOn: $model.isEnabled)
    }
}

使用美元符号 $ 运算符获取状态对象属性的 Binding。当你想要创建双向连接时，请使用绑定。在上面的代码中， Toggle 通过绑定控制模型的 isEnabled 值。

使用外部数据初始化状态对象​

当状态对象的初始状态取决于来自其容器外部的数据时，你可以从其容器的初始值设定项内显式调用该对象的初始值设定项。例如，假设上一个示例中的数据模型在初始化期间采用 name 输入，并且你希望使用来自视图外部的该名称的值。你可以通过在为视图创建的显式初始化程序内调用状态对象的初始化程序来完成此操作：

struct MyInitializableView: View {
    @StateObject private var model: DataModel

    init(name: String) {
        // SwiftUI ensures that the following initialization uses the
        // closure only once during the lifetime of the view, so
        // later changes to the view's name input have no effect.
        _model = StateObject(wrappedValue: DataModel(name: name))
    }

    var body: some View {
        VStack {
            Text("Name: \(model.name)")
        }
    }
}

执行此操作时请务必小心。 SwiftUI 仅在你第一次在给定视图中调用其初始值设定项时初始化状态对象。这确保了即使视图的输入发生变化，对象也能提供稳定的存储。但是，如果显式初始化状态对象，可能会导致意外行为或不需要的副作用。

在上面的示例中，如果 MyInitializableView 的 name 输入发生更改，SwiftUI 将使用新值重新运行视图的初始值设定项。但是，SwiftUI 仅在你第一次调用状态对象的初始值设定项时运行你提供给状态对象的初始值设定项的自动闭包，因此模型存储的 name 值不会更改。

当对象所依赖的外部数据对于对象容器的给定实例没有更改时，显式状态对象初始化效果很好。例如，你可以创建两个具有不同常量名称的视图：

var body: some View {
    VStack {
        MyInitializableView(name: "Ravi")
        MyInitializableView(name: "Maria")
    }
}
警告

即使对于可配置的状态对象，你仍然将其声明为私有。这确保你不会意外地通过视图的成员初始化程序设置参数，因为这样做可能会与框架的存储管理发生冲突并产生意外的结果。

通过更改视图标识强制重新初始化​

如果你希望 SwiftUI 在视图输入更改时重新初始化状态对象，请确保视图的标识同时更改。实现此目的的一种方法是使用 id(_:) 修饰器将视图的标识绑定到更改的值。例如，你可以确保 MyInitializableView 实例的标识在其 name 输入更改时发生更改：

MyInitializableView(name: name)
    .id(name) // Binds the identity of the view to the name property.
备注

如果你的视图出现在 ForEach 内，它会隐式接收使用相应数据元素的标识符的 id(_:) 修饰器。

如果你需要视图根据多个值的更改重新初始化状态，你可以使用 Hasher 将这些值组合成一个标识符。例如，如果你希望在 name 或 isEnabled 的值发生更改时更新 MyInitializableView 中的数据模型，则可以将这两个变量合并到单个哈希中:

var hash: Int {
    var hasher = Hasher()
    hasher.combine(name)
    hasher.combine(isEnabled)
    return hasher.finalize()
}

然后将组合哈希应用到视图作为标识符：

MyInitializableView(name: name, isEnabled: isEnabled)
    .id(hash)

请注意每次输入更改时重新初始化状态对象的性能成本。此外，更改视图标识可能会产生副作用。例如，如果视图的标识同时发生变化，SwiftUI 不会自动为视图内的更改设置动画。此外，更改标识会重置视图持有的所有状态，包括你管理为 State、FocusState、GestureState 等的值。

ObservedObject​

一种属性包装类型，它订阅可观察对象并在可观察对象发生更改时使视图无效。

public protocol ObservableObject : AnyObject {

    /// The type of publisher that emits before the object has changed.
    associatedtype ObjectWillChangePublisher : Publisher = ObservableObjectPublisher
            where Self.ObjectWillChangePublisher.Failure == Never

    /// A publisher that emits before the object has changed.
    var objectWillChange: Self.ObjectWillChangePublisher { get }
}
/// A publisher that publishes changes from observable objects.
@available(iOS 13.0, macOS 10.15, tvOS 13.0, watchOS 6.0, *)
final public class ObservableObjectPublisher : Publisher {

    /// The kind of values published by this publisher.
    public typealias Output = Void

    /// The kind of errors this publisher might publish.
    ///
    /// Use `Never` if this `Publisher` does not publish errors.
    public typealias Failure = Never

    /// Creates an observable object publisher instance.
    public init()

    /// Attaches the specified subscriber to this publisher.
    ///
    /// Implementations of ``Publisher`` must implement this method.
    ///
    /// The provided implementation of ``Publisher/subscribe(_:)-4u8kn``calls this method.
    ///
    /// - Parameter subscriber: The subscriber to attach to this ``Publisher``, after which it can receive values.
    final public func receive<S>(subscriber: S) where S : Subscriber, S.Failure == Never, S.Input == ()

    /// Sends the changed value to the downstream subscriber.
    final public func send()
}

概述​

当输入为 ObservableObject 并且你希望视图在对象的已发布属性更改时更新时，将 @ObservedObject 属性添加到 SwiftUI View 的参数中。你通常这样做是为了将 StateObject 传递到子视图中。

以下示例将数据模型定义为可观察对象，在视图中将模型实例化为状态对象，然后将该实例作为观察对象传递到子视图：

class DataModel: ObservableObject {
    @Published var name = "Some Name"
    @Published var isEnabled = false
}


struct MyView: View {
    @StateObject private var model = DataModel()

    var body: some View {
        Text(model.name)
        MySubView(model: model)
    }
}


struct MySubView: View {
    @ObservedObject var model: DataModel

    var body: some View {
        Toggle("Enabled", isOn: $model.isEnabled)
    }
}

当可观察对象的任何已发布属性发生更改时，SwiftUI 会更新依赖于该对象的任何视图。子视图还可以对模型属性进行更新，例如上面示例中的 Toggle，这些属性会传播到整个视图层次结构中的其它观察者。

备注

不要用 @ObservedObject 包装符合 Observable 协议的对象。 SwiftUI 自动跟踪主体中使用的 Observable 对象的依赖关系，并在数据更改时更新依赖视图。尝试用 @ObservedObject 包装 Observable 对象可能会导致编译器错误，因为它要求其包装的对象符合 ObservableObject 协议。

如果视图需要绑定到其主体中 Observable 对象的属性，请使用 Bindable 属性包装器包装该对象；例如， @Bindable var model: DataModel。有关更多信息，请参阅 管理应用程序中的模型数据。
