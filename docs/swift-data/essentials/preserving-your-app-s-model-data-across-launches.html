在启动过程中保留应用程序的模型数据

警告

该文档目前仅完成 Google 翻译，需进一步校对和润色。

原文：Preserving your app’s model data across launches

使用框架的宏向 SwiftData 描述你的模型类，并存储这些模型的实例，以便它们在应用程序运行时之外存在。

概述​

大多数应用程序定义了许多自定义类型，对其创建或使用的数据进行建模。例如，旅行应用程序可能会定义代表旅行、航班和预订住宿的类。使用 SwiftData，你可以快速有效地保留该数据，以便在应用程序启动时可用，并利用该框架与 SwiftUI 的集成来重新获取该数据并将其显示在屏幕上。

根据设计，SwiftData 补充了你现有的模型类。该框架提供了宏和属性包装器等工具，使你能够在 Swift 代码中明确地描述应用程序的架构，从而消除对模型和迁移映射文件等外部依赖项的任何依赖。

将类转变为模型，使其具有持久性​

要让 SwiftData 保存模型类的实例，请导入框架并使用 Model() 宏注释该类。该宏根据 PersistentModel 协议更新类，SwiftData 使用该协议检查类并生成内部架构。此外，该宏通过添加对 Observable 协议的一致性来启用类的更改跟踪。

import SwiftData

// Annotate new or existing model classes with the @Model macro.
@Model
class Trip {
    var name: String
    var destination: String
    var startDate: Date
    var endDate: Date
    var accommodation: Accommodation?
}

默认情况下，SwiftData 包含类的所有非计算属性，只要它们使用兼容类型。该框架支持基本类型，例如 Bool 、 Int 和 String ，以及复杂值类型，例如结构、枚举和其他符合标准的值类型。到 Codable 协议。

你编写的用于定义模型类的代码现在充当应用程序模型层的事实来源，框架使用它来保持持久数据处于一致状态。

自定义模型属性的持久化行为​

属性是 SwiftData 管理的模型类的属性。在大多数情况下，框架的属性默认行为就足够了。但是，如果你需要更改 SwiftData 处理特定属性持久性的方式，请使用提供的架构宏之一。例如，你可能希望通过指定属性的值在该模型的所有实例中是唯一的来避免模型数据中的冲突。

要自定义属性的行为，请使用 Attribute(_:originalName:hashModifier:) 宏注释该属性，并指定驱动所需行为的选项值：

@Attribute(.unique) var name: String

除了强制执行唯一约束之外， @Attribute 还支持保留已删除的值、Spotlight 索引和加密等功能。如果你想在基础模型数据中保留原始名称，你还可以使用 @Attribute 宏来正确处理重命名的属性。

当模型包含其类型也是模型（或模型集合）的属性时，SwiftData 会隐式地为你管理这些模型之间的关系。默认情况下，在删除相关模型实例后，框架将关系属性设置为 nil 。要指定不同的删除规则，请使用 Relationship(::originalName:inverse:hashModifier:) 宏注释该属性。例如，你可能希望在删除行程时删除所有相关住宿。有关删除规则的更多信息，请参阅 Relationship/DeleteRule-swift.enum 。

@Relationship(.cascade) var accommodation: Accommodation?

默认情况下，SwiftData 会保留模型的所有非计算属性，但你可能并不总是希望这种情况发生。例如，一类中的一个或多个属性可能只包含不需要保存的临时数据，例如即将到来的旅行目的地的当前天气。在这种情况下，使用 Transient() 宏注释这些属性，SwiftData 不会将它们的值写入磁盘。

@Transient var destinationWeather = Weather.current()

配置模型存储​

在 SwiftData 检查你的模型并生成所需的架构之前，你需要在运行时告诉它要保留哪些模型，以及（可选）用于底层存储的配置。例如，你可能希望在运行测试时存储仅存在于内存中，或者在跨设备同步模型数据时使用特定的 CloudKit 容器。

要设置默认存储，请使用 modelContainer(for:inMemory:isAutosaveEnabled:isUndoEnabled:onSetup:) 视图修饰器（或等效场景）并指定要保留的模型类型数组。如果你使用视图修饰器，请将其添加到视图层次结构的最顶部，以便所有嵌套视图继承正确配置的环境：

import SwiftUI
import SwiftData

@main
struct TripsApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
                .modelContainer(for: [
                    Trip.self,
                    Accommodation.self
                ])
        }
    }
}

如果你不使用 SwiftUI，请使用适当的初始值设定项手动创建模型容器：

import SwiftData

let container = try ModelContainer([
    Trip.self,
    Accommodation.self
])
提示

如果模型类型包含关系，你可以从数组中省略目标模型类型。 SwiftData 自动遍历模型的关系并为你包含任何目标模型类型。

或者，使用 ModelConfiguration 创建自定义存储。该类型提供了许多配置选项，包括是否：

  • 存储仅存在于内存中。
  • 该存储是只读的。
  • 该应用程序使用特定的应用程序组来存储其模型数据。
let configuration = ModelConfiguration(isStoredInMemoryOnly: true, allowsSave: false)

let container = try ModelContainer(
    for: Trip.self, Accommodation.self,
    configurations: configuration
)
警告

自动 iCloud 同步依赖于 CloudKit 权利的存在，并且 SwiftData 使用它在该权利中找到的第一个容器。如果你的应用需要特定容器，请使用 ModelConfiguration 实例来指定该容器。

保存模型供以后使用​

要在运行时管理模型类的实例，请使用模型上下文 - 负责内存中模型数据并与模型容器协调以成功保留该数据的对象。要获取绑定到主要参与者的模型容器的上下文，请使用 modelContext 环境变量：

import SwiftUI
import SwiftData

struct ContentView: View {
    @Environment(\.modelContext) private var context
}

在视图之外，或者如果你不使用 SwiftUI，请直接使用模型容器访问相同的 actor 绑定上下文：

let context = container.mainContext

在这两种情况下，返回的上下文都会定期检查它是否包含未保存的更改，如果是，则代表你隐式保存这些更改。对于手动创建的上下文，请将 autosaveEnabled 属性设置为 true 以获得相同的行为。

要使 SwiftData 能够持久保存模型实例并开始跟踪对其的更改，请将实例插入上下文中：

var trip = Trip(name: name,
                destination: destination,
                startDate: startDate,
                endDate: endDate)

context.insert(trip)

插入后，你可以通过调用上下文的 save() 方法立即保存，或者依赖上下文的隐式保存行为。上下文自动跟踪对其已知模型实例的更改，并将这些更改包含在后续保存中。除了保存之外，你还可以使用上下文来获取、枚举和删除模型实例。有关详细信息，请参阅 ModelContext 。

获取模型以进行显示或附加处理​

开始保留模型数据后，你可能需要检索该数据，具体化为模型实例，并在视图中显示这些实例或对它们执行其他操作。 SwiftData 提供了 Query 属性包装器和 FetchDescriptor 类型来执行提取。

要获取模型实例，并可选择应用搜索条件和首选排序顺序，请在 SwiftUI 视图中使用 @Query 。 @Model 宏为模型类添加了 Observable 一致性，使 SwiftUI 能够在任何获取的实例发生更改时刷新包含视图。

import SwiftUI
import SwiftData

struct ContentView: View {
    @Query(sort: \.startDate, order: .reverse) var allTrips: [Trip]

    var body: some View {
        List {
            ForEach(allTrips) {
                TripView(for: $0)
            }
        }
    }
}

在视图之外，或者如果你不使用 SwiftUI，请使用 ModelContext 上的两个 fetch 方法之一。每个方法都需要一个包含谓词和排序顺序的 FetchDescriptor 实例。获取描述符允许影响批处理、偏移和预取等的附加配置。

let context = container.mainContext

let upcomingTrips = FetchDescriptor<Trip>(
    predicate: #Predicate { $0.startDate > Date.now },
    sort: \.startDate
)
upcomingTrips.fetchLimit = 50
upcomingTrips.includePendingChanges = true

let results = context.fetch(upcomingTrips)

有关谓词的更多信息，请参阅 Predicate 。
