## 4.4 线程组地址空间

GPU 计算单元可以在 _threadgroup_ 中同时执行多个线程，并且 GPU 可以为其每个计算单元执行单独的线程组。

线程组中的线程可以通过共享线程组内存中的数据来协同工作，这在大多数设备上比共享设备内存中的数据更快。使用线程组地址空间可以：

- 在内核、网格或对象函数中分配线程组变量。 - 定义一个内核、片段或对象函数参数，它是指向线程组地址的指针。

请参阅 Metal 功能集表，了解哪些 GPU 支持片段着色器的线程组空间参数。

内核、网格或对象函数中的线程组变量仅在执行内核的线程组的生命周期内存在。中间渲染内核函数中的线程组变量在图块上的中间渲染和片段内核函数中是持久的。

此示例内核演示了如何在线程组地址空间中声明变量和参数。 （下面代码中的 [[threadgroup]] 属性在第 5.2.1 节中进行了解释。）

kernel void my_kernel(threadgroup float \*sharedParameter [[threadgroup(0)]], ...) { // 在线程组地址空间中分配一个 float。线程组浮动 sharedFloat；

// 在线程组地址空间中分配一个包含 10 个浮点数的数组。线程组浮点 sharedFloatArray[10]； ... }

## 4.4.1 SIMD 组和 Quad 组

macOS：自 Metal 2 起支持 SIMD 组。自 Metal 2.1 起支持四组。

iOS：自 Metal 2.2 起支持一些 SIMD 组函数。从 Metal 2 开始就支持四组。

在线程组中，你可以将线程划分为 _SIMD-groups_ ，它们是并发执行的线程的集合。到 SIMD 组的映射在内核执行期间、具有相同启动参数的给定内核的分派以及分派内从一个线程组到另一个线程组（不包括存在不一致线程组大小的后沿线程组）中保持不变）。此外，线程组内的所有 SIMD 组都需要具有相同的大小，除了具有最大索引的 SIMD 组之外，如果线程组的大小不能被 SIMD 的大小整除，则索引可能更小。组。

_quad-group_ 是线程执行宽度为 4 的 SIMD 组。
