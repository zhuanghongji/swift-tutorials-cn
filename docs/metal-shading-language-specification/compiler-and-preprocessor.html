编译器和预处理器

原文：1.5 Compiler and Preprocessor

你可以在线使用 Metal 编译器（使用适当的 API 来编译 Metal 源代码）或离线使用。你可以使用适当的 Metal API 加载离线编译为二进制文件的 Metal 源。

本节介绍 Metal 编译器支持的编译器选项，并将它们分类为预处理器选项、数学内在函数选项、控制优化的选项、杂项编译选项和链接。

1.5.1 预处理器编译器选项​

以下选项控制实际编译之前在每个程序源上运行的 Metal 预处理器：

-D 姓名

将 name 预定义为宏，定义为 1。

-D 名称=定义

``` 定义的内容被标记和处理，就像它们出现在 #define 指令中一样。此选项允许你编译 Metal 代码以启用或禁用功能。你可以多次使用此选项，预处理器将按照定义出现的顺序处理它们。````

  • 我目录

``` 将目录 dir 添加到头文件目录的搜索路径中。该选项仅适用于离线编译器。````

1.5.2 预处理器定义​

Metal 编译器默认设置了许多预处理器定义，包括：

METAL_VERSION // 设置为 Metal 语言版本

METAL_MACOS // 如果使用 macOS Metal 语言编译则设置

METAL_IOS // 如果使用 iOS Metal 语言编译则设置

METAL // 如果使用统一 Metal 语言编译则设置 // 使用 - std=metal3.0 或更高版本设置。

你可以使用定义有条件地应用仅在更高版本的语言版本中可用的着色语言功能（请参阅控制语言版本的编译器选项）。

版本号为 MajorMinorPatch。例如，对于 Metal 1.2，补丁 0，METAL_VERSION 为 120；对于 Metal 2.1，补丁 1，METAL_VERSION 是 211。

要有条件地包含使用 Metal 2 中引入的功能的代码，你可以在代码中使用预处理器定义，如下所示：

#if METAL_VERSION >= 200 // 需要 Metal 2 中引入的功能的代码。#endif

1.5.3 数学内在函数编译器选项​

这些选项控制编译器有关浮点运算的行为，在速度和正确性之间进行权衡。

  • ffast-math（默认） - fno-fast-math

有关数学函数的更多信息，请参阅第 6.5 节。有关普通和快速数学函数的相对误差的更多信息，请参阅第 7.4 节。

这些选项启用（默认）或禁用可能违反 IEEE 754 标准的浮点算术优化。它们还启用或禁用单精度浮点标量和向量类型的数学函数的高精度变体。

浮点运算的快速数学优化包括：

  • 无 NaN：允许优化假设参数和结果不是 NaN（“不是数字”）。- 无 INF：允许优化假设参数和结果不是正无穷大或负无穷大。- 无符号零：允许优化将零参数或结果的符号视为无关紧要。- 允许倒数：允许优化使用参数的倒数而不是执行除法。- 允许重新关联：允许代数等效转换，例如重新关联可能会显着改变浮点结果的浮点运算。

  • 允许合约：允许跨语句进行浮点收缩。例如，允许将一个乘数和一个附加值融合成一个融合乘加。

在程序中使用快速数学时，重要的是要了解编译器可以假设某些属性并相应地进行优化。例如，使用快速数学断言着色器永远不会生成 Inf 或 NaN。如果程序有表达式 X/Y，编译器可以假设 Y 永远不为零，因为这可能会导致正/负无限或 NaN，具体取决于 X 的值。如果 Y 可以为零，则开发人员将得到一个未定义的值如果用快速数学编译程序。

默认情况下，编译器允许浮点收缩。例如，a*b+c 可以转换为单个融合乘加。如果其他表达式没有收缩，这些收缩可能会导致计算差异。要禁止编译器收缩，请传递以下选项：

-ffp-合同=关闭

编译器还支持使用程序来禁用收缩：

#pragma STDC FP_CONTRACT OFF

1.5.4 不变性编译器选项​

如果你使用支持 iOS 14 或 macOS 11 的 SDK 进行构建，则需要传递以下选项以支持顶点不变性。

  • f 保持不变性

``` 在顶点着色器中保留用 [[invariant]] 标记的计算的不变性。如果未设置，则忽略 [[invariant]]。````

在 Metal 的早期版本中，[[invariant]] 是一种尽力而为的分析，用于标记哪些操作需要保持不变，并且在某些情况下可能会失败。这被保守的不变模型所取代，其中编译器标记不进入不变计算的操作。这将保证任何不变的计算保持不变。此选项可能会降低性能，因为它可能会阻止某些优化以保持不变性。

1.5.5 优化编译器选项​

这些选项控制编译器的优化级别。

-O

``` 优化性能（默认）。````

  • 操作系统

与 O2 一样，进行了额外的优化以减少代码大小。

1.5.6 最大总线程组大小选项​

所有操作系统：自 Metal 3 起。

此选项指定翻译单元中每个函数的线程组中的线程数（值）。

  • fmax-total-threads-per-threadgroup=<值>

第 5.1.3 节、第 5.1.7 节和第 5.1.8 节中描述的属性 [[max_total_threads_per_threadgroup]] 函数属性优先于编译选项。

此选项对于设置选项以使为动态库编译的函数与 PSO 兼容非常有用。

1.5.7 纹理写入舍入模式​

通过将 -ftexture-write-rounding-mode 编译器标志设置为表 1. 1 中的选项之一，配置纹理写入浮点像素类型的舍入模式。

表 1. 1. 舍入模式

舍入模式 描述

本机（默认）

``` 纹理写入使用硬件的本机舍入策略。````

rte

``` 所有操作系统：自 Metal 2.3 起。````

``` 纹理四舍五入到最接近的偶数。````

rtz

``` 所有操作系统：自 Metal 2.3 起。````

`` 纹理朝零方向四舍五入写入。````

-ftexture-write-rounding-mode 标志可用于以下 SDK：

  • macOS 11 或更高版本 - iOS 14 或更高版本

有关哪些 GPU 系列支持本机以外的舍入模式的更多信息，请参阅 Metal 功能集表。

1.5.8 启用模块的编译器选项​

编译器支持多个选项来控制模块的使用。这些选项仅适用于离线编译器：

-f 模块

`` 启用模块功能。````

  • f 隐式模块映射

启用对名为 module.modulemap 或类似名称的模块映射文件的隐式搜索。默认情况下，-fmodules 启用此选项。（编译器选项 - fno-implicit-module-maps 禁用此选项。）

  • fno 隐式模块映射

``` 禁用对名为 module.modulemap 的模块映射文件的隐式搜索。仅当使用 -fmodule-map-file 显式指定或由另一个模块映射文件传递使用时，才会加载模块映射文件。````

  • fmodules-cache-path=<目录>

``` 指定模块缓存的路径。如果未提供，编译器将选择适合系统的默认值。````

  • fmodule-map-file=<文件>

如果加载了指定模块映射文件的目录或其子目录之一的标头，则加载指定的模块映射文件。````

如果你使用支持 iOS 16 或 macOS 13 的 SDK 进行构建，-fmodules 具有以下附加选项。

  • fmodules=[mode] 支持的模式值为：stdlib：启用模块功能，但将模块映射的搜索限制为 Metal 标准库。默认情况下，使用支持 iOS 16 或 macOS 13 的 SDK 启用。

`` all：启用模块功能（相当于-fmodules）。none：禁用模块功能。````

1.5.9 控制语言版本的编译器选项​

以下选项控制编译器接受的统一图形和计算语言的版本：

-标准=

确定要使用的语言版本。必须提供此选项的值，该值必须是以下之一：

  • ios-metal1.0：支持统一图形和计算语言修订版 1。适用于 iOS 8 的程序。[[已弃用并将在未来的操作系统中删除]]

  • ios-metal1.1：支持统一图形和计算语言修订版 1.适用于 iOS 9 的程序。- ios-metal1.2：支持统一图形和计算语言修订版 1.适用于 iOS 10 的程序。- ios-metal2.0 ：支持 iOS 11 统一图形和计算语言修订版 2.程序。- ios-metal2.1：支持 iOS 12 统一图形和计算语言修订版 2.程序。- ios-metal2.2：支持统一图形和计算语言修订版。计算语言修订版 2. iOS 13 的 2 个程序。- ios-metal2.3：支持统一图形和计算语言修订版 2。iOS 14 的 3 个程序。- ios-metal2.4：支持统一图形和计算语言修订版 2 4 个适用于 iOS 15 的程序。- macos-metal1,1 或 osx-metal1.1：支持 macOS 10.11 的统一图形和计算语言修订版 1.1 程序。- macos-metal1.2 或 osx-metal1.2：支持 macOS 10.12 的统一图形和计算语言修订版 1.2 程序。- macos-metal2。0 或 osx-metal2.0：支持 macOS 10.13 的统一图形和计算语言修订版 2.0 程序。- macos-metal2.1：支持 macOS 10.14 的统一图形和计算语言修订版 2.1 程序。- macos-metal2.2：支持 macOS 10.1 5 的统一图形和计算语言修订版 2. 2 程序。- macos-metal2.3：支持 macOS 11.0 的统一图形和计算语言修订版 2. 3 程序 - macos-metal2.4：支持 macOS 12.0 的统一图形和计算语言修订版 2.4 程序。

请注意，macos-* 在 macOS10.13 SDK 及更高版本中可用。

从 macOS13.0、iOS 16 和 tvOS 16 开始，Metal 已经在平台之间统一了着色语言。

  • Metal 3.0：支持 macOS 13 .0、iOS 16 和 tvOS 16 的统一图形和计算语言修订版 3.0 程序。- Metal 3.1：支持 macOS 14 .0 的统一图形和计算语言修订版 3.1 程序、iOS 17 和 tvOS 17。

可以使用以下选项：

  • 错误

``` 将所有警告变成错误。````

-w

`` 禁止所有警告消息。````

1.5.11 目标条件​

Metal 定义了几个宏，可用于确定着色器运行的平台。以下宏在 中定义：

TARGET_OS_MAC ：生成的代码将在 Mac OS X 变体下运行 TARGET_OS_OSX ：生成的代码将在 OS X 设备下运行 TARGET_OS_IPHONE ：为固件、设备或模拟器生成的代码 TARGET_OS_IOS ：生成的代码将在 iOS 下运行 TARGET_OS_TV ：生成的代码将在 Apple TV 操作系统下运行 TARGET_OS_MACCATALYST ：生成的代码将在 MacOS 下运行 TARGET_OS_SIMULATOR ：生成的代码将在模拟器下运行

请注意，此标头不是 的一部分。

1.5.12 动态库链接器选项​

Metal 编译器驱动程序可以将选项传递给链接器。以下是其中一些选项的简要说明。有关详细信息，请参阅 Metal 链接器。

  • 动态库

``` 指定输出是动态库。````

  • 安装名称

`` 与 -dynamiclib 一起使用来指定加载程序期望安装和找到动态库的位置。与@executable_path 和@loader_path 一起使用。````

1.5.13 编译为 GPU 二进制文件的选项​

如果你使用支持 iOS 16 或 macOS 13 的 SDK 进行构建，则可以使用以下选项编译为 GPU 二进制文件。

  • arch [架构] 指定要构建的架构。
  • gpu-family [gpu 系列名称] 指定与要构建的 MTLGPUFamily 关联的体系结构。请参阅 Metal API 中的 MTLGPUFamily 以获取可用系列的列表。
  • N [descriptor.mtlp-json] 以 Metal 脚本格式指定管道描述符。描述符文件必须以 .mtlp-json 结尾。

1.5.14 生成 Metal 库符号文件的选项​

如果你使用支持 iOS 15 或 macOS 12 的 SDK 进行构建，则可以使用以下选项来生成 Metal 库符号文件。

  • frecord-sources 使编译器能够将源信息存储到 AIR 或 Metal 库文件 (.metallib) 中。
  • frecord-sources=flat 如果生成 AIR 文件，则启用编译器存储源信息。如果生成 Metal 库文件，则启用编译器将源信息存储在符号伴随文件 (.metallibsym) 中。

有关更多信息，请参阅 developer.apple.com 上的生成和加载 Metal 符号文件。
