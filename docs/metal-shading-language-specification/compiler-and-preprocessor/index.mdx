## 1.5 编译器和预处理器

你可以在线使用 Metal 编译器（使用适当的 API 来编译 Metal 源代码）或离线使用。你可以使用适当的 Metal API 加载离线编译为二进制文件的 Metal 源。

本节介绍 Metal 编译器支持的编译器选项，并将它们分类为预处理器选项、数学内在函数选项、控制优化的选项、杂项编译选项和链接。

## 1.5.2 预处理器定义

Metal 编译器默认设置了许多预处理器定义，包括：

**METAL_VERSION** // 设置为 Metal 语言版本

**METAL_MACOS** // 如果使用 macOS Metal 语言编译则设置

**METAL_IOS** // 如果使用 iOS Metal 语言编译则设置

**METAL** // 如果使用统一 Metal 语言编译则设置 // 使用 - std=metal3.0 或更高版本设置。

你可以使用定义有条件地应用仅在更高版本的语言版本中可用的着色语言功能（请参阅控制语言版本的编译器选项）。

版本号为 MajorMinorPatch。例如，对于 Metal 1.2，补丁 0，**METAL_VERSION** 为 120；对于 Metal 2.1，补丁 1，**METAL_VERSION** 是 211。

要有条件地包含使用 Metal 2 中引入的功能的代码，你可以在代码中使用预处理器定义，如下所示：

#if **METAL_VERSION** >= 200 // 需要 Metal 2 中引入的功能的代码。#endif

## 1.5.3 数学内在函数编译器选项

这些选项控制编译器有关浮点运算的行为，在速度和正确性之间进行权衡。

- ffast-math（默认） - fno-fast-math

有关数学函数的更多信息，请参阅第 6.5 节。有关普通和快速数学函数的相对误差的更多信息，请参阅第 7.4 节。

这些选项启用（默认）或禁用可能违反 IEEE 754 标准的浮点算术优化。它们还启用或禁用单精度浮点标量和向量类型的数学函数的高精度变体。

浮点运算的快速数学优化包括：

- 无 NaN：允许优化假设参数和结果不是 NaN（“不是数字”）。 - 无 INF：允许优化假设参数和结果不是正无穷大或负无穷大。 - 无符号零：允许优化将零参数或结果的符号视为无关紧要。 - 允许倒数：允许优化使用参数的倒数而不是执行除法。 - 允许重新关联：允许代数等效转换，例如重新关联可能会显着改变浮点结果的浮点运算。

``2023 - 06 - 02 |版权所有 © 2023 苹果公司 |版权所有。 ````

- 允许合约：允许跨语句进行浮点收缩。例如，允许将一个乘数和一个附加值融合成一个融合乘加。

在程序中使用快速数学时，重要的是要了解编译器可以假设某些属性并相应地进行优化。例如，使用快速数学断言着色器永远不会生成 Inf 或 NaN。如果程序有表达式 X/Y，编译器可以假设 Y 永远不为零，因为这可能会导致正/负无限或 NaN，具体取决于 X 的值。如果 Y 可以为零，则开发人员将得到一个未定义的值如果用快速数学编译程序。

默认情况下，编译器允许浮点收缩。例如，a\*b+c 可以转换为单个融合乘加。如果其他表达式没有收缩，这些收缩可能会导致计算差异。要禁止编译器收缩，请传递以下选项：

-ffp-合同=关闭

编译器还支持使用程序来禁用收缩：

`#pragma STDC FP_CONTRACT OFF`

## 1.5.4 不变性编译器选项

如果你使用支持 iOS 14 或 macOS 11 的 SDK 进行构建，则需要传递以下选项以支持顶点不变性。

- f 保持不变性

``` 在顶点着色器中保留用 [[invariant]] 标记的计算的不变性。如果未设置，则忽略 [[invariant]]。 ````

在 Metal 的早期版本中，[[invariant]] 是一种尽力而为的分析，用于标记哪些操作需要保持不变，并且在某些情况下可能会失败。这被保守的不变模型所取代，其中编译器标记不进入不变计算的操作。这将保证任何不变的计算保持不变。此选项可能会降低性能，因为它可能会阻止某些优化以保持不变性。

## 1.5.11 目标条件

Metal 定义了几个宏，可用于确定着色器运行的平台。以下宏在 中定义：

TARGET_OS_MAC ：生成的代码将在 Mac OS X 变体下运行 TARGET_OS_OSX ：生成的代码将在 OS X 设备下运行 TARGET_OS_IPHONE ：为固件、设备或模拟器生成的代码 TARGET_OS_IOS ：生成的代码将在 iOS 下运行 TARGET_OS_TV ：生成的代码将在 Apple TV 操作系统下运行 TARGET_OS_MACCATALYST ：生成的代码将在 MacOS 下运行 TARGET_OS_SIMULATOR ：生成的代码将在模拟器下运行

请注意，此标头不是 的一部分。
