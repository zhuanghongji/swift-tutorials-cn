## 5.7 图形功能-签名匹配

图形函数签名是图形函数的输入或输出的参数列表。

## 5.7.1 顶点——片段签名匹配

你可以在顶点函数和片段函数之间传递两种数据：用户定义的变量和内置变量。

你可以使用 [[stage_in]] 属性声明片段函数的每个实例输入。这些是由关联的顶点函数输出的。

你可以使用第 5.2.3 节中定义的属性之一来声明内置变量。使用这些属性的变量示例有：

- 顶点函数输出（带有 [[position]]、[[point_size]] 或 [[clip_distance]] 属性）， - 光栅器输出（带有 [[point_coord]]、[[front_face]]、[[[[sample_id]] 或 [[sample_mask]] 属性）， - 或引用帧缓冲区颜色值的片段函数输入（带有 [[color]]）。

始终返回指定 [[position]] 属性的内置变量。对于具有 [[point_size]] 或 [[clip_distance]] 属性的内置变量，该属性还必须指定相应的顶点函数输出。如果在片段函数中使用和读取它们，则着色器具有未定义的行为。

你还可以声明作为光栅化器输出的内置变量，或使用适当的属性引用帧缓冲区颜色值作为片段函数输入。

你还可以使用属性 [[user(name)]] 语法为任何用户定义的变量指定属性名称。

如果满足以下条件，则顶点函数和片段函数具有匹配的签名：

- 片段函数中声明的 [[stage_in]] 属性没有输入参数。 - 对于用 [[stage_in]] 声明的片段函数参数，与该参数关联的类型中的每个元素可以是以下之一：由光栅化器生成的内置变量、作为输入传递给片段的帧缓冲区颜色值函数，或用户生成的顶点函数输出。对于光栅化器或帧缓冲区颜色值生成的内置变量，不需要将匹配类型与顶点返回类型的元素相关联。对于用户生成的输出元素，适用以下规则：

如果使用 [[user(name)]] 为元素指定属性名称，则属性名称必须与顶点函数返回类型中的元素匹配。如果不指定 [[user(name)]] 属性名称，则参数名称和类型必须匹配。在任何一种情况下，它们相应的数据类型也必须匹配，或者片段函数参数类型需要为 interpolant，其中 T 是顶点返回类型中元素的类型。

## 5.7.2 Mesh——片段签名匹配

你可以将网格 的顶点 (V) 和图元 (P) 的两种数据从网格函数传递到片段函数：用户定义的变量和内置变量。顶点 (V) 中定义的每顶点网格输出始终进行插值，而图元 (P) 中定义的每图元网格输出从不进行插值。由于这种差异，用户生成的输出的签名匹配规则已根据第 5.7.1 节“顶点 - 片段签名匹配”中描述的规则进行了调整。

如果满足以下条件，则给定片段输入*匹配*用户从顶点 (V) 和图元 (P) 生成的网格输出：如果使用 [[user(name)]] 为元素指定属性名称，则属性名称必须与网格输出的返回类型中的元素匹配。如果不指定 [[user(name)]] 属性名称，则参数名称和类型必须匹配。无论哪种情况，它们相应的数据类型也必须匹配，或者片段函数参数类型需要为 interpolant，其中 T 是顶点返回类型中元素的类型。

如果满足以下条件，则网格函数和片段函数具有用户生成的输入与用户生成的网格输出的匹配签名：

- 对于给定的用户生成的具有平坦插值的片段输入： o 有匹配的每个基元网格输出，并且输出无需插值即可传播到片段输入。

``` o 有一个匹配的每顶点网格输出，并且激发顶点的输出无需插值即可传播到片段输入。 ````

- 对于给定的用户生成的具有非平坦插值的片段输入：

``` o 有一个匹配的每个基元网格输出，并且输出无需插值即可传播到片段输入。 o 有一个匹配的每顶点网格输出，并且该输出以与非平坦顶点输出插值相同的方法跨图元进行插值。 ````
