# 模式

<DocCaution />

<OriginalLink
  title="匹配和解构值"
  value="https://docs.swift.org/swift-book/documentation/the-swift-programming-language/patterns"
/>

模式表示单个值或复合值的结构。例如，元组的结构(1, 2)是两个元素的逗号分隔列表。因为模式代表值的结构而不是任何一个特定值，所以您可以将它们与各种值匹配。例如，模式(x, y)匹配元组(1, 2)和任何其他双元素元组。除了将模式与值匹配之外，您还可以提取部分或全部复合值并将每个部分绑定到常量或变量名称。

在 Swift 中，有两种基本的模式：一种可以成功匹配任何类型的值，另一种可能在运行时无法匹配指定的值。

第一种模式用于解构简单变量、常量和可选绑定中的值。这些包括通配符模式、标识符模式以及包含它们的任何值绑定或元组模式。您可以为这些模式指定类型注释，以限制它们仅匹配特定类型的值。

第二种模式用于完整模式匹配，您尝试匹配的值在运行时可能不存在。这些包括枚举案例模式、可选模式、表达式模式和类型转换模式。您可以在语句的 case 标签 switch、语句 catch 的子句或, , , or -语句 do 的 case 条件中使用这些模式。ifwhileguardforin

:::note 模式语法

```
pattern → wildcard-pattern type-annotation?

pattern → identifier-pattern type-annotation?

pattern → value-binding-pattern

pattern → tuple-pattern type-annotation?

pattern → enum-case-pattern

pattern → optional-pattern

pattern → type-casting-pattern

pattern → expression-pattern
```

:::

## 通配符模式

通配符模式匹配并忽略任何值并由下划线 ( \_) 组成。当您不关心要匹配的值时，请使用通配符模式。例如，以下代码遍历封闭范围 1...3，在循环的每次迭代中忽略范围的当前值：

```swift
for _ in 1...3 {
    // Do something three times.
}
```

:::note 通配符模式的语法

```
wildcard-pattern → _
```

:::

## 标识符模式

标识符模式匹配任何值并将匹配的值绑定到变量或常量名称。例如，在以下常量声明中，是一个与 type 的值相匹配的标识符模式：someValue42Int

```swift
let someValue = 42
```

当匹配成功时，该值 42 被绑定（分配）到常量 name 。someValue

当变量或常量声明左侧的模式是标识符模式时，标识符模式隐含地是值绑定模式的子模式。

:::note 标识符模式的语法

```
identifier-pattern → identifier
```

:::

## 值绑定模式

值绑定模式将匹配的值绑定到变量或常量名称。将匹配值绑定到常量名称的值绑定模式以 let 关键字开头；绑定到变量名称的那些以 var 关键字开头。

值绑定模式中的标识符模式将新的命名变量或常量绑定到它们的匹配值。例如，您可以分解元组的元素并将每个元素的值绑定到相应的标识符模式。

```swift
let point = (3, 2)
switch point {
// Bind x and y to the elements of point.
case let (x, y):
    print("The point is at (\(x), \(y)).")
}
// Prints "The point is at (3, 2)."
```

在上面的示例中，let 分发到元组模式中的每个标识符模式(x, y)。由于此行为，switch 个案 `case let (x, y):` 和 `case (let x, let y):` 匹配相同的值。

:::note 值绑定模式的语法
value-binding-pattern → var pattern | let pattern
:::

## 元组模式

元组模式是以逗号分隔的零个或多个模式列表，括在括号中。元组模式匹配相应元组类型的值。

您可以使用类型注释来约束元组模式以匹配特定种类的元组类型。例如，(x, y): (Int, Int)常量声明中的元组模式 let (x, y): (Int, Int) = (1, 2)仅匹配两个元素均为 type 的元组类型 Int。

当元组模式用作 for-in 语句或变量或常量声明中的模式时，它只能包含通配符模式、标识符模式、可选模式或包含这些模式的其他元组模式。例如，以下代码无效，因为 0 元组模式中的元素(x, 0)是表达式模式：

```swift
let points = [(0, 0), (1, 0), (1, 1), (2, 0), (2, 1)]
// This code isn't valid.
for (x, 0) in points {
    /* ... */
}
```

包含单个元素的元组模式周围的括号无效。该模式匹配该单个元素类型的值。例如，以下是等效的：

```swift
let a = 2        // a: Int = 2
let (a) = 2      // a: Int = 2
let (a): Int = 2 // a: Int = 2
```

:::note 元组模式的语法

```
tuple-pattern → ( tuple-pattern-element-list? )

tuple-pattern-element-list → tuple-pattern-element | tuple-pattern-element , tuple-pattern-element-list

tuple-pattern-element → pattern | identifier : pattern
```

:::

## 枚举案例模式

枚举案例模式匹配现有枚举类型的案例。枚举案例模式出现在语句案例标签和、、和-语句 switch 的案例条件中。ifwhileguardforin

如果您尝试匹配的枚举案例有任何关联值，则相应的枚举案例模式必须指定一个元组模式，其中每个关联值都包含一个元素。switch 有关使用语句匹配包含关联值的枚举案例的示例，请参阅关联值。

枚举案例模式还匹配包装在可选中的该案例的值。这种简化的语法使您可以省略可选模式。请注意，因为 Optional 是作为枚举实现的，.none 并且.some 可以出现在与枚举类型的情况相同的开关中。

```swift
enum SomeEnum { case left, right }
let x: SomeEnum? = .left
switch x {
case .left:
    print("Turn left")
case .right:
    print("Turn right")
case nil:
    print("Keep going straight")
}
// Prints "Turn left"
```

:::note 枚举格模式语法

```
enum-case-pattern → type-identifier? . enum-case-name tuple-pattern?
```

:::

## 可选图案

可选模式匹配 some(Wrapped)枚举案例中包装的值 `Optional<Wrapped>` 。可选模式由紧跟问号的标识符模式组成，并出现在与枚举案例模式相同的位置。

因为可选模式是 Optional 枚举案例模式的语法糖，所以以下是等价的：

```swift
let someOptional: Int? = 42
// Match using an enumeration case pattern.
if case .some(let x) = someOptional {
    print(x)
}

// Match using an optional pattern.
if case let x? = someOptional {
    print(x)
}
```

for 可选模式提供了一种方便的方法来迭代-语句中的可选值数组 in，仅对非元素执行循环体 nil。

```swift
let arrayOfOptionalInts: [Int?] = [nil, 2, 3, nil, 5]
// Match only non-nil values.
for case let number? in arrayOfOptionalInts {
    print("Found a \(number)")
}
// Found a 2
// Found a 3
// Found a 5
```

:::note 可选模式的语法

```
optional-pattern → identifier-pattern ?
```

:::

## 类型转换模式

有两种类型转换模式，ispattern 和 aspattern。该 is 模式仅出现在 switch 陈述案例标签中。和 is 模式 as 具有以下形式：

```swift
is <#type#>
<#pattern#> as <#type#>
```

如果某个值 `is` 在运行时的类型与模式右侧指定的类型相同，is 或者该类型的子类，则该模式匹配该值。该 is 模式的行为类似于 `is` 运算符，因为它们都执行类型转换但丢弃返回的类型。

如果某个值 as 在运行时的类型与模式右侧指定的类型相同，as 或者该类型的子类，则该模式匹配该值。如果匹配成功，则将匹配值的类型强制转换为模式右侧指定的模式 as。

switch 有关使用语句将值 is 与模式匹配的示例 as，请参阅 Any 和 AnyObject 的类型转换。

:::note 类型转换模式的语法

```
type-casting-pattern → is-pattern | as-pattern

is-pattern → is type

as-pattern → pattern as type
```

:::

## 表情模式

表达式模式表示表达式的值。表达式模式仅出现在 switch 语句案例标签中。

使用 Swift 标准库运算符将表达式模式表示的表达式与输入表达式的值进行比较~=。~=如果运算符返回 ，则匹配成功 true。默认情况下，~=运算符使用运算符比较相同类型的两个值==。它还可以通过检查值是否包含在该范围内来匹配具有值范围的值，如以下示例所示。

```swift
let point = (1, 2)
switch point {
case (0, 0):
    print("(0, 0) is at the origin.")
case (-2...2, -2...2):
    print("(\(point.0), \(point.1)) is near the origin.")
default:
    print("The point is at (\(point.0), \(point.1)).")
}
// Prints "(1, 2) is near the origin."
```

您可以重载~=运算符以提供自定义表达式匹配行为。例如，您可以重写上面的示例以将 point 表达式与点的字符串表示形式进行比较。

```swift
// Overload the ~= operator to match a string with an integer.
func ~= (pattern: String, value: Int) -> Bool {
    return pattern == "\(value)"
}
switch point {
case ("0", "0"):
    print("(0, 0) is at the origin.")
default:
    print("The point is at (\(point.0), \(point.1)).")
}
// Prints "The point is at (1, 2)."
```

:::note 表达式模式的语法

```
expression-pattern → expression
```

:::
