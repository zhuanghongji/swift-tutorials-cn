向现有类型添加功能

原文：https://docs.swift.org/swift-book/documentation/the-swift-programming-language/extensions


扩展为现有类、结构、枚举或协议类型添加新功能。这包括扩展你无权访问原始源代码的类型的能力（称为追溯建模）。扩展类似于 Objective-C 中的类别。（与 Objective-C 类别不同，Swift 扩展没有名称。）

Swift 中的扩展可以：

  • 添加计算实例属性和计算类型属性
  • 定义实例方法和类型方法
  • 提供新的初始化器
  • 定义下标
  • 定义和使用新的嵌套类型
  • 使现有类型符合协议

在 Swift 中，你甚至可以扩展协议以提供其要求的实现或添加符合类型可以利用的额外功能。有关详细信息，请参阅协议扩展。

备注

扩展可以为类型添加新功能，但不能覆盖现有功能。

扩展语法​

使用关键字声明扩展 extension ：

extension SomeType {
    // new functionality to add to SomeType goes here
}

扩展可以扩展现有类型，使其采用一种或多种协议。要添加协议一致性，你可以像为类或结构编写协议名称一样编写协议名称：

extension SomeType: SomeProtocol, AnotherProtocol {
    // implementation of protocol requirements goes here
}

以这种方式添加协议一致性在 Adding Protocol Conformance with an Extension 中有所描述。

扩展可用于扩展现有的通用类型，如扩展通用类型中所述。你还可以扩展泛型类型以有条件地添加功能，如使用泛型 Where 子句的扩展中所述。

备注

如果你定义扩展以向现有类型添加新功能，则新功能将在该类型的所有现有实例上可用，即使它们是在定义扩展之前创建的。

计算属性​

扩展可以将计算实例属性和计算类型属性添加到现有类型。此示例将五个计算实例属性添加到 Swift 的内置 Double 类型，以提供对使用距离单位的基本支持：

extension Double {
    var km: Double { return self * 1_000.0 }
    var m: Double { return self }
    var cm: Double { return self / 100.0 }
    var mm: Double { return self / 1_000.0 }
    var ft: Double { return self / 3.28084 }
}
let oneInch = 25.4.mm
print("One inch is \(oneInch) meters")
// Prints "One inch is 0.0254 meters"
let threeFeet = 3.ft
print("Three feet is \(threeFeet) meters")
// Prints "Three feet is 0.914399970739201 meters"

这些计算属性表示一个 Double 值应该被认为是一个特定的长度单位。虽然它们是作为计算属性实现的，但可以使用点语法将这些属性的名称附加到浮点文字值，作为使用该文字值执行距离转换的一种方式。

在此示例中，Double 值被认为表示“一米”。这就是计算属性返回的原因——表达式被认为计算了 的值。1.0mself1.mDouble1.0

其它单位需要一些转换才能表示为以米为单位测量的值。一公里与 1,000 米相同，因此 km 计算的属性将值乘以转换为以米表示的数字。同样，一米有 3.28084 英尺，因此计算属性将基础值除以，将其从英尺转换为米。1_000.00ftDouble3.28084

这些属性是只读的计算属性，因此 get 为简洁起见，它们没有关键字表示。它们的返回值是 类型，并且可以在接受 Doublea 的任何地方的数学计算中使用：Double

let aMarathon = 42.km + 195.m
print("A marathon is \(aMarathon) meters long")
// Prints "A marathon is 42195.0 meters long"
备注

扩展可以添加新的计算属性，但不能添加存储属性，也不能向现有属性添加属性观察器。

初始化器​

扩展可以向现有类型添加新的初始值设定项。这使你能够扩展其它类型以接受你自己的自定义类型作为初始化参数，或提供未包含在该类型的原始实现中的其它初始化选项。

扩展可以向类添加新的便利初始化器，但不能向类添加新的指定初始化器或反初始化器。指定的初始化器和反初始化器必须始终由原始类实现提供。

如果你使用扩展将初始化器添加到为其所有存储属性提供默认值且未定义任何自定义初始化器的值类型，则可以从扩展的初始化器中调用该值类型的默认初始化器和成员初始化器. 如果你将初始化程序编写为值类型的原始实现的一部分，情况就不会如此，如“值类型的初始化程序委派”中所述。

如果你使用扩展将初始化程序添加到在另一个模块中声明的结构中，则新的初始化程序无法访问，self 直到它从定义模块调用初始化程序。

下面的示例定义了一个自定义 Rect 结构来表示一个几何矩形。Size 该示例还定义了两个名为 and 的支持结构 Point，它们都为其所有属性提供默认值：0.0

struct Size {
    var width = 0.0, height = 0.0
}
struct Point {
    var x = 0.0, y = 0.0
}
struct Rect {
    var origin = Point()
    var size = Size()
}

因为该结构为其所有属性提供默认值，所以它会自动接收默认初始化程序和成员初始化程序，如默认初始化程序 Rect 中所述。这些初始化器可用于创建新实例：Rect

let defaultRect = Rect()
let memberwiseRect = Rect(origin: Point(x: 2.0, y: 2.0),
    size: Size(width: 5.0, height: 5.0))
你可以扩展该Rect结构以提供一个额外的初始化器，该初始化器采用特定的中心点和大小：

extension Rect {
    init(center: Point, size: Size) {
        let originX = center.x - (size.width / 2)
        let originY = center.y - (size.height / 2)
        self.init(origin: Point(x: originX, y: originY), size: size)
    }
}

这个新的初始化器首先根据提供的 center 点和 size 值计算一个合适的原点。然后初始化程序调用结构的自动成员初始化程序，它将新的原点和大小值存储在适当的属性中：init(origin:size:)

let centerRect = Rect(center: Point(x: 4.0, y: 4.0),
    size: Size(width: 3.0, height: 3.0))
// centerRect's origin is (2.5, 2.5) and its size is (3.0, 3.0)
备注

如果你提供一个带有扩展的新初始化器，你仍然有责任确保在初始化器完成后每个实例都被完全初始化。

方法​

扩展可以向现有类型添加新的实例方法和类型方法。下面的示例添加了一个调用 repetitions 该 Int 类型的新实例方法：

extension Int {
    func repetitions(task: () -> Void) {
        for _ in 0..<self {
            task()
        }
    }
}

该 repetitions(task:) 方法采用 type 的单个参数 () -> Void，表示一个没有参数且不返回值的函数。

定义此扩展后，你可以对 repetitions(task:) 任何整数调用该方法以执行多次任务：

3.repetitions {
    print("Hello!")
}
// Hello!
// Hello!
// Hello!

改变实例方法​

添加了扩展的实例方法也可以修改（或改变）实例本身。修改 self 或其属性的结构和枚举方法必须将实例方法标记为 mutating，就像从原始实现中改变方法一样。

下面的示例添加了一个新的 mutating 方法，调用 squareSwift 的 Int 类型，它对原始值进行平方：

extension Int {
    mutating func square() {
        self = self * self
    }
}
var someInt = 3
someInt.square()
// someInt is now 9

下标​

扩展可以向现有类型添加新的下标。此示例将整数下标添加到 Swift 的内置 Int 类型。此下标[n]返回数字 n 右侧的小数位：

  • 123456789[0] 返回 9
  • 123456789[1] 返回 8

… 等等：

extension Int {
    subscript(digitIndex: Int) -> Int {
        var decimalBase = 1
        for _ in 0..<digitIndex {
            decimalBase *= 10
        }
        return (self / decimalBase) % 10
    }
}
746381295[0]
// returns 5
746381295[1]
// returns 9
746381295[2]
// returns 2
746381295[8]
// returns 7

如果该 Int 值没有足够的数字用于请求的索引，则下标实现返回 0，就好像数字已经用零填充到左边一样：

746381295[9]
// returns 0, as if you had requested:
0746381295[9]

嵌套类型​

扩展可以向现有类、结构和枚举添加新的嵌套类型：

extension Int {
    enum Kind {
        case negative, zero, positive
    }
    var kind: Kind {
        switch self {
        case 0:
            return .zero
        case let x where x > 0:
            return .positive
        default:
            return .negative
        }
    }
}

此示例将新的嵌套枚举添加到 Int. 此枚举称为 Kind，表示特定整数表示的数字类型。具体来说，它表示数字是负数、零还是正数。

Int 此示例还向 中添加了一个名为 的新计算实例属性 kind，该属性返回 Kind 该整数的适当枚举案例。

嵌套枚举现在可以与任何值一起使用 Int：

func printIntegerKinds(_ numbers: [Int]) {
    for number in numbers {
        switch number.kind {
        case .negative:
            print("- ", terminator: "")
        case .zero:
            print("0 ", terminator: "")
        case .positive:
            print("+ ", terminator: "")
        }
    }
    print("")
}
printIntegerKinds([3, 19, -27, 0, -6, 0, 7])
// Prints "+ + - 0 - 0 + "

此函数接受一个值的输入数组并依次迭代这些值。对于数组中的每个整数，该函数会考虑该整数的计算属性，并打印适当的描述。printIntegerKinds(_:)Intkind

备注

number.kind 已知属于 类型。因此，所有 case 值都可以在语句中以速记形式书写，例如 rather than 。Int.KindInt.Kindswitch.negativeInt.Kind.negative
